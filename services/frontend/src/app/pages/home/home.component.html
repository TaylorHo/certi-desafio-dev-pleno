Gerenciamento de Usuários

Bem vindo(a) ao projeto de Gerenciamento de Usuários! Existem 3 cargos/funções disponíveis aos usuários (Super Admin, Admin e Cliente). Além disso, essa plataforma conta com 6 páginas (Login, Registro, Página Inicial, Lista de Usuários, Erro 401 e Erro 404).

Segurança

Usuários recém registrados recebem a função de "Cliente" por padrão e, por questões de segurança, um usuário recém criado, que é um cliente, não deveria ter acesso à listagem, edição, criação e exclusão de usuários e, por isso, se você está logado como um usuário "Cliente" (provavelmente logo após o registro), você não terá acesso à isso.

Para edição, exclusão e listagem de usuários, você pode logar como Admin ou Super Admin utilizando as credenciais abaixo:

  • Super Admin:
    • Usuário: superadmin@mail.com
    • Senha: password
  • Admin:
    • Usuário: admin@mail.com
    • Senha: password

Regras de Negócio

  • Todos os usuários podem ver a página inicial quando logam em suas contas. A página inicial é protegida para permitir apenas usuários autorizados. Se o usuário não tem um token de acesso, ele é redirecionado para a página de login.
  • Seguindo a mesma lógica, todos os endpoints do backend relacionados ao CRUD de usuários são pretegidos. Apenas os endpoints de login e de registro não requerem autenticação prévia.
  • Se um usuário com a função "Cliente" tentar acessar a página com a lista de usuários, ele será redirecionado para a página de erro 401 (Proibido).
  • Funções de editar, criar, excluir e listar usuários estão disponíveis na página da lista de usuários
  • O nome de usuário não pode ser alterado após o registro, pois é usado para login.
  • Se um usuário Admin ou Super Admin editar seu cargo e mudá-lo para "Cliente", ele perdem o acesso à página com a lista de usuários assim que tentar fazer qualquer outra ação.
  • Um usuário do tipo Admin pode apenas alterar informações de usuários do tipo "Cliente", e não de outros usuários do tipo "Admin" e muito menos "Super Admin".
  • O token JWT utilizado na autenticação expira após 1 hora.

Funções

Sobre essa Plataforma

  • Frontend criado em Angular 15
  • Backend criado em NestJS
  • Documentação da API de usuários acessível em {{usersApiURL}}
  • Biblioteca de estilos Material Design
  • Sass como template engine de CSS
  • Autenticação com JWT
  • Banco de dados NoSql com mongoDB
  • Messageria com Kafka para gerenciamento de usuários em forma de fila
  • Ambientes de desenvolvimento, desenvolvimento em container, staging e produção
  • Load Balancing com NGINX (quando subido o ambiente de produção)

Considerações finais

É tudo uma questão de experiência, empenho e dedicação ao aprendizado, não de tempo.
